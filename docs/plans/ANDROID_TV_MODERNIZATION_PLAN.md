This document is now a historical reference only.

## Why this file changed

The repo had multiple overlapping TV planning documents:

- `TV_ROADMAP.md`
- `ANDROID_TV_MODERNIZATION_PLAN.md`
- `docs/features/tv detail.md`
- `docs/features/TV_DETAIL_UPGRADES_REVIEW.md`
- `docs/features/TV_SHOWS_SCREEN_IMPROVEMENTS.md`

That split caused status drift. In particular, this file marked several Android TV areas as fully complete, while the broader roadmap still treated focus, detail polish, playback polish, and device validation as open work.

To keep one unified execution document, all active TV planning now lives in `TV_ROADMAP.md`.

## How to use this file now

- Read it only as background on the first TV modernization pass.
- Do not update checklists here.
- Put all new TV planning/status changes into `TV_ROADMAP.md`.
